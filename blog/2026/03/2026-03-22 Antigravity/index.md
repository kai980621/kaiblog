## Antigravity 是什麼？

簡單說，它是一個 **AI 版的結對程式設計夥伴**（pair programmer）。
跟一般你可能聽過的 ChatGPT 之類的不一樣——你不是在跟它聊天，它是真的**直接幫你動檔案、跑指令、看你的 code 在幹嘛**。

比如說你只要跟它說「幫我把這個功能改一下」，它不是給你一個需要你複製貼上的程式碼，而是**自己就把你的 `.js` 改好了**，還會告訴你它做了什麼、為什麼這樣改。

有點像是你旁邊坐了一個工程師，但這個工程師不會跟你說「你自己試試看」😂

## 另類的 VS Code

講到這個我覺得蠻有趣的。

Antigravity 本身**就是一個編輯器**，長得跟 VS Code 幾乎一模一樣——左邊是你的檔案結構、右邊是程式碼、底下是 terminal。如果你原本用 VS Code，大概 0 秒就能上手介面。

但它跟 VS Code 最不一樣的地方在於：**它旁邊有一個 AI 是真的知道你整個專案在幹嘛的**。

一般你在 VS Code 裡用 GitHub Copilot 之類的東西，AI 大概只看得到你目前打開的這個檔案。
但 Antigravity 是真的**把你整個專案的架構、所有檔案之間的關係都搞清楚了**，然後才幫你做事。

所以你跟它說「把這個 bug 修掉」，它不會只改你打開的這一個檔案，它知道問題可能出在別的地方，**自己去找、自己去改**。

某種程度上，Antigravity 就是 VS Code + 一個真的聽得懂你說什麼的 AI，套在一起的結果。

## 我是怎麼開始用的

其實一開始我根本不知道這個東西的存在。

某天在弄我的 blog（就是你現在看的這個）的時候，遇到一個問題一直解不掉，Google 了很久沒找到答案，後來嘗試用了 Antigravity 問問看。

結果它直接就找到問題出在哪裡，幫我修好了，過程大概 **2 分鐘**。

那個當下我就知道這東西不一樣。

## 跟 ChatGPT 的差別

說說我的親身比較：

- **ChatGPT**：你要把 code 複製貼上給它，它給你回答，你再貼回去，錯了再問一次。~~有時候貼到手痠~~
- **Antigravity**：它直接看你的專案、知道你的目錄長怎樣、幫你改好、問你 OK 不 OK

最大的差別就是**它知道你整個專案的上下文**，不是只看你貼給它的那一段。所以它給的答案比較不會答非所問。

## 用下來有什麼缺點？

說真的，不是完美的。

有時候它太積極，你說「幫我看看這裡」，它可能會「順便」把旁邊的東西也改一改，
這種時候你要確認一下它有沒有動到你不想讓它動的地方。

另外，複雜一點的問題偶爾還是需要來回幾次才能解決，但這很正常就是了，又不是真的神。
